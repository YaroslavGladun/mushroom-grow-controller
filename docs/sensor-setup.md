# Підключення та монтаж датчика SHT31-D

Практичний посібник: як фізично під'єднати датчик t°/RH **SHT31-D** до
**ESP32-WROOM-32 (DevKit V1)**, яким кабелем вивести його в камеру та **як закріпити**
всередині контейнера. Доповнює загальний `build-plan.md`.

> **Припущення (позначено явно):**
> - ESP32 живе в **герметичному боксі ЗЗОВНІ** контейнера; у камеру заходить лише зонд
>   датчика крізь гермоввід.
> - Використовується модуль SHT31-D з монтажним отвором (напр. Adafruit-стиль).
> - Живлення й логіка датчика — **3.3 В** (без рівневого перетворювача).

---

## 1. Що потрібно (матеріали)

| Позиція | Призначення |
| --- | --- |
| Модуль **SHT31-D** (I²C breakout) | Датчик t°/RH |
| **4 дроти-перемички «мама–мама» (DuPont F–F)** | Швидке з'єднання на столі (тест) |
| **Екранована кручена пара, 4 жили, 0.5–1 м** | Постійний вивід у камеру |
| Резистори підтяжки **2.2–4.7 кОм** (×2) | На SDA/SCL, якщо на модулі їх немає |
| Гермоввід (PG7/PG9) | Герметичний ввід кабелю в бокс/стінку |
| **Нейлонові гвинт M2.5–M3 + стійки (standoff)** | Кріплення плати (не сталь — іржавіє) |
| Пластиковий кутник / смужка пластику | Кронштейн для підвісу |
| Нейтральний (оксимний) силікон / акрил conformal | Захист плати від конденсату |

> ⚠️ Уся **металева** фурнітура в камері (95 % RH) кородує — беріть **пластик/нейлон**.
> Не звичайні DuPont-перемички для постійного монтажу в камеру, а екранований кабель.

---

## 2. Розпіновка та з'єднання

| Пін модуля SHT31-D | Пін ESP32-WROOM-32 (DevKit V1) |
| --- | --- |
| **VIN / VCC** | **3V3** (вихід регулятора 3.3 В) |
| **GND** | **GND** |
| **SDA** | **GPIO21** (D21, SDA за замовч.) |
| **SCL / SCK** | **GPIO22** (D22, SCL за замовч.) |
| **ADR** | → GND = адреса `0x44` (за замовч.) · → 3.3 V = `0x45` |
| ALR/ALERT | не підключати (опційно) |

> ⚠️ **Живлення датчика — з піна `3V3`, НЕ з `VIN`/`5V`.** ESP32 і його GPIO **не
> толерантні до 5 В**: 5 В на датчику/шині пошкодить і датчик, і ESP32. Рівневий
> перетворювач не потрібен — усе на 3.3 В.

> **Примітка:** фізична розкладка пінів на платах DevKit V1 різних виробників
> відрізняється — звіряйтеся з **шовкографією** плати (підписи `3V3`, `GND`, `D21`,
> `D22`). SDA/SCL в ESP32 можна ремапити на інші GPIO у прошивці, але за замовчуванням
> це GPIO21 / GPIO22.

---

## 3. Кабель: довжина і тип

- **Ціль: 0.5–1 м, і не більше ~1 м.** I²C — коротка шина; після ~1 м зростає ризик збоїв.
- **Як міряти:** відстань бокс → крізь стінку → точка датчика **+ 10–15 см** на петлю-крапельник і провис. Не тягнути «внатяг». Ставте бокс близько до контейнера.
- **Тип:** екранована кручена пара, 4 жили; екран заземлити **з одного боку** (на GND ESP32).
- Швидкість шини **100 кГц** для кабелю в камеру: задається в прошивці — ESPHome
  `i2c: { frequency: 100kHz }` (або Arduino `Wire.setClock(100000)`).
- Підтяжки **2.2–4.7 кОм** і сам breakout лишати **зовні**, у сухому боксі.

**Якщо ≤1 м не виходить:**
- активний I²C-розширювач (LTC4311), **або**
- окремий **DS18B20** (1-Wire) для температури (тягне кілька метрів), а SHT31 — коротким для вологості.

---

## 4. Де ставити датчик

- **На висоті плодового тіла (canopy)** — у зоні росту, не під кришкою й не біля дна.
- **Не торкаючись стінок** — на них конденсат, буде хибних 99 %.
- **Подалі від прямого струменя туману** з трубки — інакше миттєві хибні 100 %.
- **Мембраною донизу (або вертикально)** — щоб краплі стікали, а не лежали на мембрані.
- Так, щоб на датчик **не капало** з кришки.

---

## 5. Як закріпити (монтаж)

**Принцип:** тримати плату **за монтажний отвір/край пластиком**, мембраною вниз,
**підвішено від кришки або верхньої стінки**, щоб датчик висів у середині об'єму.

### Рекомендований спосіб — кронштейн + нейлонові стійки

- [ ] Прикрутити плату **нейлоновим гвинтом M2.5–M3 через монтажний отвір** до пластикового кутника/смужки (через нейлонові стійки)
- [ ] Закріпити кутник на **кришці або верхній частині бічної стінки**, щоб плата звисала вниз на висоту canopy, **мембраною донизу**
- [ ] Вивести кабель угору до гермовводу, лишити **петлю-крапельник**
- [ ] За потреби — маленький пластиковий **козирок/дашок** над датчиком проти крапель із кришки

### Простіші альтернативи

| Спосіб | Як | Коли доречно |
| --- | --- | --- |
| Стяжки до опори | Пластиковий пруток/трубку на стінку, плату притягнути 2 стяжками **за краї** | Найдешевше |
| Жорсткий ввід | Кабель у **напівжорсткій трубці** крізь гермоввід сам тримає датчик на кінці | Мінімум деталей |
| Перфорований тримач | Пластиковий стакан/коробочка з отворами, у ній плата | Якщо з кришки капає |

### Обов'язкові деталі

- Мембрана **вниз**; датчик — в об'ємі повітря, **не на поверхні** з конденсатом.
- Кріплення **пластик/нейлон**, не сталь.
- Силікон для фіксації — **лише нейтральний (оксимний)**, **нічого на мембрану/чіп**, клеїти за край.
- Плату перед монтажем **покрити conformal coating**, мембрану лишити відкритою.
- **Не** гарячим клеєм по платі; стяжкою не перетискати компоненти.

## 6. Налаштувати I²C і перевірити

### ESPHome (рекомендовано)

Мінімальний фрагмент конфігу — шина I²C + датчик SHT31:

```yaml
i2c:
  sda: GPIO21
  scl: GPIO22
  frequency: 100kHz
  scan: true            # у лозі з'явиться знайдена адреса (0x44 або 0x45)

sensor:
  - platform: sht3xd
    address: 0x44        # 0x45, якщо ADR → 3.3 В
    heater_max_duty: 0.05  # ~5% — вбудований нагрівач підсушує мембрану (0.0 = вимк)
    update_interval: 30s
    temperature:
      name: "Камера t°"
    humidity:
      name: "Камера RH"
```

Прошити (перший раз — по USB, далі OTA) і подивитися лог:

```bash
esphome run mushroom.yaml       # прошивка
esphome logs mushroom.yaml      # scan: true покаже пристрій на 0x44 (або 0x45) + t°/RH
```

### Arduino (альтернатива)

```cpp
#include <Wire.h>
#include "Adafruit_SHT31.h"
Adafruit_SHT31 sht31 = Adafruit_SHT31();

void setup() {
  Serial.begin(115200);
  Wire.begin(21, 22);           // SDA=GPIO21, SCL=GPIO22
  Wire.setClock(100000);        // 100 кГц для кабелю в камеру
  if (!sht31.begin(0x44)) {     // 0x45 — якщо ADR на 3.3 В
    Serial.println("SHT31 не знайдено — перевір I²C/адресу/живлення 3V3");
    while (1) delay(10);
  }
}

void loop() {
  Serial.printf("t=%.1f C  RH=%.1f %%\n",
                sht31.readTemperature(), sht31.readHumidity());

  // Імпульс вбудованого нагрівача — випарити конденсат у вологій камері:
  sht31.heater(true);
  delay(3000);
  sht31.heater(false);
  delay(30000);
}
```

> **Порада:** періодично зганяйте конденсат вбудованим нагрівачем SHT31. В ESPHome —
> невеликий `heater_max_duty` (авто-підсушування), в Arduino/MicroPython — ручний
> імпульс `heater(true)` на кілька секунд що 30–60 хв.
